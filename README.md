# Guide Questions
1. Why is removing duplicates important in data analysis?
 Duplicates cause inflated totals and inaccurate counts, leading to unreliable reports and wrong business decisions.

2. What problems can missing values cause in reports?
Missing values can cause calculation errors, incomplete visuals, and misleading results in reports.

3. How does correct data typing affect calculations?
Wrong data types prevent calculations from working — for example, numbers stored as text can't be summed or averaged.

4. What naming conventions improve data readability?
Consistent, lowercase names like order_id make columns easier to read, reference, and understand across the team.


# Guide Questions

1. What is the difference between Merge and Append?
Merge combines tables side by side using a matching column (like a JOIN), while Append stacks tables on top of each other to add more rows.

2. Why use Inner Join instead of Left Join?
 Inner Join only keeps records that match in both tables, giving cleaner and more accurate data. Left Join keeps all records from the left table even if there's no match, which can result in null values.

3. What happens to unmatched records in an Inner Join?
 Unmatched records are completely removed — only rows with matching values in both tables are kept.

4. When is Append more appropriate than Merge?
Append is better when you have multiple tables with the same columns and just want to combine all their rows together, like monthly sales files.
